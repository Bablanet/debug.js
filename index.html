Debug.js

Debugging javascript is hard. debug.js is will make it easier. It is a bunch tools which make javascript easier to debug.

Learn more »

Features

  • Strong type checking
  • variable value bound checking
  • garbage collector
  • NaN detection
    • You do lot of Math ? Like in 3d with WebGL ?
    • tired of computations ending with NaN and not knowing where it comes from ?
    • Debug.js can help you. Just use the strong type checking
    • It will automatically detect when a property becomes NaN and notify you immediatly.
    • PropertyAttr.define(foo, 'bar')
              .typeCheck(['nonan', Number])
  • Monitor who allocate object
  • assert which actually stop excecution
    • It is often best when you are notified immediatly if a issue occurs.
    • But surprisingly, default console.assert() doesn't stop the execution in browser.
    • Assert is stopping the programm execution in other language, but not in javascript.
    • If you find that unfortunate, debug.js provides a fix.
    • With assertWhichStop.js, you can trigger an exception or even trigger js debugger as soon as an assertion is false.
    • It is even backward compatible. It is as simple as 
    • assertWhichStop.overloadConsole();
  • Private Property and Function
    • You handle a large code base ?
    • You wish to ensure that what is supposed to be private is actually private
    • debug.js can help you.
    • With privateforjs.js, you can define which functions are allowed to access a given property, or call a function.
    • aClass = FunctionAttr.define(aClass).privatize().done();
  • function attribute
  • Function call wrapper
  • queuable getter setter
  • stack extraction
